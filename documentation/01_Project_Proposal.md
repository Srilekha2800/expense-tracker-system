# 1. Introduction

The Expense Tracker System is a responsive web application designed to help users manage their personal finances effectively. The application allows users to track their income, expenses, savings, and financial activities through an interactive dashboard and detailed reports.

The project is initially developed using HTML5, CSS3, and JavaScript. In future versions, the application will be enhanced with Java, Spring Boot backend and MySQL database to provide secure authentication and persistent data storage.By integrating the frontend with a Spring Boot backend and a MySQL database, the application will evolve into a complete full-stack web application. This integration will enable secure user authentication, persistent storage of financial transactions, and dynamic generation of financial reports, providing users with a more interactive and reliable experience
---

# 2. Problem Statement

Many individuals find it difficult to monitor their daily financial transactions and savings. Traditional methods such as notebooks and spreadsheets are often time-consuming, difficult to maintain, and lack real-time financial insights. Physical records are also susceptible to loss or damage, making long-term financial tracking challenging.

There is a need for a simple and user-friendly software application that enables users to track expenses, analyze spending habits, and manage personal finances efficiently.

---

# 3. Project Objectives

The main objectives of this project are:

- To develop a responsive expense tracking web application.
- To provide a user-friendly interface for managing financial transactions.
- To allow users to record and monitor their income and expenses.
- To display financial summaries through dashboards and reports.
- To improve understanding of frontend web development concepts.
- To prepare the project for future integration with Spring Boot and MySQL.

# 10. Assumptions

The initial version of the project assumes that:

- Users access the application through a modern web browser.
- Internet connectivity is not required for frontend functionality using Local Storage.
- Backend services are not available during the frontend development phase.
- Future backend integration will replace Local Storage with a MySQL database.
